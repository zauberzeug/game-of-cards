## 2026-05-09: decision recorded

Same-repo (deck on mainline of the code repo) is the only configuration that ships as supported. Sibling-repo, submodule, and hosted SaaS are documented as 'possible but unsupported' — users can wire them up themselves, but GoC commits to the same-repo experience. — Each alternative adds substantial path-resolution and discovery code for a persona (OSS contributor base) that is not yet validated. The active epic `support-external-game-of-cards-state-location` already explores deck-path indirection; better to let that mature than ship a half-supported sibling-repo discovery now. Submodule (UX friction) and hosted SaaS (own epic — `explore-saas-deck-hosting-with-optional-tracker-sync`) are deferred. Reconsider when an OSS user actually asks.. Gate session → none.
