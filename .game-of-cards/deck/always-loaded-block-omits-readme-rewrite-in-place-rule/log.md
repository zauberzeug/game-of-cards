## 2026-06-20 — filed and implemented

Motivated by a recurring downstream failure: agents append a
`## Correction` / `## Update` section to a card's `README.md` while
leaving the now-refuted claim standing above it, so the card asserts
two contradicting verdicts and the stale top-framing misleads the next
reader.

Investigation found the discipline was **not** missing — it was only
on opt-in surfaces:

- `card-schema/SKILL.md` "What goes where" already names the exact
  antipattern ("a new 'Latest finding (DATE)' block at the bottom of
  the README … accumulates contradicting versions of the truth").
- `advance-card/SKILL.md` disproved flow says "rewrite … body".
- `deck/SKILL.md` Layout block carries the dashboard/journal framing.
- Tooling enforcement exists only for `goc decide` (the reminder + the
  `DECISION_CONTRADICTS_VERDICT` advisory validator).

All of those require loading a skill or taking a verb. The failure
happens during free-form `README.md` correction edits that do neither,
so the only reliably-loaded surface — the `<!-- BEGIN GOC -->` block
in `AGENTS.md` / `CLAUDE.md` — is where the rule had to land.

### Why the always-loaded block, not another skill body

This extends the trajectory the family already set. The predecessor
[clarify-readme-as-dashboard-and-log-md-as-history](../clarify-readme-as-dashboard-and-log-md-as-history/)
deliberately pushed the dashboard/journal distinction *up* from
`card-schema` into `deck/SKILL.md` so it would "land on the first read,
not only inside card-schema." The decide-path card
[goc-decide-leaves-stale-verdict-content-when-recording-a-rescope](../goc-decide-leaves-stale-verdict-content-when-recording-a-rescope/)
added tooling enforcement but scoped it to `goc decide`, and its own
body flagged the gap: the general "fix every occurrence when you demote
a claim" discipline is neither enforced nor reminded outside that verb.
This card pushes the rule one level further up — to the one surface
present in every session, regardless of skill or verb.

### Meta-pattern siblings (guidance placement)

Beyond the README-contradiction family above, this card is one instance
of a recurring GoC design tension: **where a given discipline lives
(always-loaded root block vs. load-on-demand skill) determines whether
it is present on the path where its failure occurs.** Closest prior
articulations:

- [relationship-modeling-has-no-discoverable-home](../relationship-modeling-has-no-discoverable-home/)
  — a discipline whose guidance had no surface the agent would reach
  when needed (same root shape, different discipline).
- [shrink-root-guidance-files-by-moving-content-into-skills](../shrink-root-guidance-files-by-moving-content-into-skills/)
  — the *inverse* placement call: content moved **out** of the root
  block into skills to cut context cost. This card moves one rule the
  other way. The trade-off (discoverability vs. always-loaded weight)
  is the shared axis.

No new generalization card filed: the meta-pattern is already
represented by these closed cards, and GoC files placement decisions
per concrete instance rather than as a speculative blanket audit. This
note is the connection the pattern-generalization check asks for.

### What landed

- `goc/templates/AGENTS_GOC.md` (source of truth, ships to consumers):
  new bolded block "**The README is a dashboard, not a changelog.**"
  inserted immediately before "Closure is not frozenness." so the
  general rewrite-in-place rule sits directly before its post-close
  exception; the block points forward to that exception via "(next)".
- `AGENTS.md` marker block: mirrored byte-for-byte (this repo dogfoods
  the template; `_append_marker_block` regenerates identical content on
  the next `goc upgrade`).
- `claude-plugin/`, `codex-plugin/`, `openclaw-plugin/` `goc/templates/`
  mirrors: regenerated by `scripts/sync_plugin_assets.py` (AGENTS_GOC.md
  is bundled into each plugin's vendored engine tree).

Scope guard: pure always-loaded-guidance edit. No engine code and no new
validator — the general-case tooling catch is intentionally left out of
scope; the existing `goc decide` / `validate` enforcement is named in
the block as partial coverage, not extended here.
