# OpenClaw card-schema skill mirror is stale — CI red on main

## Location

- Drifted committed file: `openclaw-plugin/skills/card-schema/SKILL.md`
- Source-of-truth template: `goc/templates/skills/card-schema/SKILL.md`
- Guard that fires: `tests/test_plugin_mirror_parity.py::OpenClawSkillPortDriftTest::test_committed_ports_match_fresh_render`

## What's broken

The OpenClaw plugin skill payload is a hand-ported mirror of the
shipped skill templates, regenerated by
`scripts/port_skills_to_openclaw.py`. Unlike the `claude-plugin/` and
`codex-plugin/` mirrors — which the `sync-plugin-assets` pre-commit
hook auto-regenerates and stages — the OpenClaw port is **deliberately
not auto-staged** (AGENTS.md: "Skills are NOT auto-synced into the
commit … the porter applies non-trivial normalization worth
eyeballing"). The drift is instead caught after the fact by a guard
test.

Two template edits landed without the accompanying manual re-port:

- `cd0cded fix(skills/card-schema): fold waiting conjuncts into not-waiting_impedes in ready predicate`
- `3427a8d fix(engine/title-guard): catch math symbols and underscores …` (added the `Underscores` title-antipattern table row)

The committed OpenClaw mirror still carries the pre-edit text. Running
the porter regenerates exactly those two regions:

```diff
 ready ⇔ status == open
       ∧ human_gate == none
-      ∧ waiting_on unset
-      ∧ (waiting_until absent or in the past)
+      ∧ not waiting_impedes(card)   # no waiting_on, OR its waiting_until has elapsed
...
+| Underscores | `my_first_card` | Lower-kebab the intent; the slug pattern allows `[a-z0-9-]` only, underscores aren't allowed |
```

## Empirical evidence

`python scripts/port_skills_to_openclaw.py --check` on the committed tree:

```text
ERROR: openclaw-plugin/skills/ have drifted from goc/templates/skills/:
  openclaw-plugin/skills/card-schema/SKILL.md
Fix: run `python scripts/port_skills_to_openclaw.py` and commit the result.
```

`uv run python -m unittest tests.test_plugin_mirror_parity`:

```text
FAIL: test_committed_ports_match_fresh_render (…OpenClawSkillPortDriftTest)
AssertionError: Lists differ: [] != ['openclaw-plugin/skills/card-schema/SKILL.md']
```

The CI step `Run regression tests` (`uv run python -m unittest discover
-s tests`) runs this test, so `main` is red and every open PR inherits
the failure until the re-port lands.

## Why it matters

A red build on `main` blocks the merge pipeline for all other work and
masks any *new* regression the same test would catch. The guard is
working as designed (it caught the drift); the defect is simply that
an edit slipped through without the manual re-port the workflow
requires.

## Fix

Run the porter and commit its output:

```bash
python scripts/port_skills_to_openclaw.py
git add -- openclaw-plugin/skills/card-schema/SKILL.md
git commit -- openclaw-plugin/skills/card-schema/SKILL.md
```

The porter is deterministic and idempotent, so this is a purely
mechanical re-sync — no `index.ts`/manifest changes expected. This card
is scoped to clearing the drift; it does **not** propose auto-staging
the OpenClaw port (that would contradict the documented
review-the-normalization design choice in AGENTS.md).
